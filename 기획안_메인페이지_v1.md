## 1. 복순도가에서 가져올 것 (실사이트 분석 결과)

boksoon.com 메인은 **풀스크린 스냅 5개 섹션**으로 구성되어 있고, 각 섹션의 공식은 동일하다:

| 요소 | 복순도가 방식 |
|---|---|
| 스크롤 | 한 휠에 한 섹션씩 넘어가는 fullpage 스냅. 우측에 도트 네비게이션 |
| 섹션 공식 | **배경(영상/사진/반반 분할) + 중앙 카피 3줄 + 영문 서브카피 + 아웃라인 버튼 1개** — 한 화면에 메시지 하나만 |
| 레이아웃 리듬 | ① 풀스크린 영상 → ② 좌사진/우단색 분할 → ③ 좌단색/우사진 분할(교차) → ④ 풀스크린 사진 → ⑤ 인스타그램 피드 + 푸터 |
| 헤더 | 좌측 로고(국문+한자), 우측 텍스트 네비 + 아이콘, 투명 배경으로 콘텐츠 위에 얹힘 |
| 타이포 | 국문 세리프(명조) 헤드라인, 흰색, 중앙 정렬. 영문 세리프 서브카피 |
| 색 | 크림/베이지 웜톤 단색면 + 자연광 사진. 채도 낮고 따뜻함 |
| CTA | 섹션마다 딱 1개, 가는 테두리의 아웃라인 버튼 (about / Subscription / Store / Experience) |

이 공식을 그대로 따르되, 내용물을 인사동보리밥의 이야기로 교체한다.

## 3. 페이지 구조 — 풀스크린 스냅 8섹션

스크롤 한 번 = 이야기 한 장. 「우리의 이야기」 6부작의 흐름을 그대로 스크롤 축으로 옮긴다.

### S1. 히어로 — "인사동 골목의 한 그릇"
- **배경**: 프리픽사진1(북촌 골목) 또는 한옥 4K 영상. 위에 은은한 어두운 오버레이
- **카피** (story6 원문 변형, 흰색 명조 중앙 3줄):
  > 서울 인사동 골목길, 지하 한 켠
  > 화려함보다는 정겨움을 담은
  > 따뜻한 밥 한 그릇
- 영문 서브카피: `Insadong Boribap — A warm bowl, made by hand.`
- CTA: 아웃라인 버튼 `우리의 이야기` (스크롤 유도)
- 헤더: 좌 로고(인사동보리밥 + INSADONG BORIBAP), 우 네비 `이야기 · 밥상 · 공간 · 창업안내 · 구매하기`

### S2. 이야기 하나 — 인사동 보리밥 (브랜드 선언)
- **레이아웃**: 복순도가 ②번 방식 — 좌측 절반 사진(보리밥 한 그릇/보리 일러스트), 우측 절반 크림 단색(한지 텍스처)
- 카피: story6 원문 — "어느 날 점심, 소박한 보리밥 한 그릇이 마음까지 든든하게…"
- **세로쓰기 헤드라인** `인사동 보리밥,` + 붉은 낙관 도장 — 스토리 카드의 시그니처를 웹에 그대로
- CTA: `브랜드 이야기 →`

### S3. 이야기 둘 — 매일 직접, 정성껏
- **레이아웃**: 좌우 반전 (좌 크림 / 우 사진 — 나물 손질·주방 손 사진, KakaoTalk 시리즈 중 선별)
- 카피: story3 원문 — "식탁에 오르는 모든 음식은 직접 하나하나 우리 손으로 만듭니다…"

### S4. 이야기 셋 — 재료의 진심 (게장 + 참기름 통합)
- **레이아웃**: 풀스크린 사진(간장게장 클로즈업) 위 카피, 또는 3분할 카드(게장/참기름/고추장)
- 카피: story1·2 축약 — "연평도 암꽃게, 황학동 김용달님의 참기름 — 재료부터 진짜를 씁니다"

### S5. 이야기 넷 — 정갈한, 맛
- **레이아웃**: 좌 사진(양념·장 사진) / 우 한지 단색
- 카피: story4 원문 — "지나치게 맵거나 달지 않고, 재료 본연의 맛과 자연스럽게 어우러지는 맛…"

### S6. 밥상 — 메뉴 쇼케이스
- **레이아웃**: 풀스크린으로 정식 한 상 사진(인사동보리밥정식.jpg 또는 GJF 상차림). 하단에 정식 4종(보리밥/불고기/제육/보쌈) 미니 카드 가로 배열
- 카피: "매일 줄 서서 먹는 그 밥상" (상페 문안 재사용)
- CTA: `온라인으로 만나기` → 네이버 스마트스토어 링크아웃

### S7. 공간 — 인사동에서
- **레이아웃**: 매장·인사동 거리 사진 (GJF/KakaoTalk 시리즈), 주소·영업시간·네이버플레이스 링크
- 카피: story5 변형 — "따뜻한 한 끼가 되도록, 마음을 담습니다"

### S8. 초대 + 푸터 — 창업 안내
- **배경**: 메인배경이미지1(어두운 그레인) — 사이트에서 유일한 다크 섹션으로 무게감 전환
- 카피: "이 밥상을 함께 차릴 분을 찾습니다" + 그래이츠 신뢰 요소 한 줄(HACCP 자체공장 · 수출 브랜드)
- CTA: `창업 문의` (문의 페이지/폼으로)
- 푸터: 인스타그램 피드 스트립(@계정), 회사 정보, ⓒ GREATS

### 타이포그래피 (가이드: "상페 결 + 좀 더 웜한 폰트")
- **헤드라인**: 국문 세리프 — 1순위 **마루 부리**, 2순위 본명조(Noto Serif KR). 스토리 카드의 붓 명조 느낌과 가장 가깝고 웜함
- **본문/UI**: Pretendard (얇은 웨이트로 모던함 유지)
- **포인트**: 세로쓰기(`writing-mode: vertical-rl`) 헤드라인을 S2 등 1~2곳에만 — 과용 금지
- ※ 산돌구름 설치돼 있음 → 보유 라이선스 폰트 확인 후 확정

### 모티프
- 한옥 지붕 라인 일러스트(상페에서 추출) — 섹션 구분 장식
- 붉은 낙관 도장 — 브랜드 서명처럼 스토리 섹션마다 1개
- 수채화 일러스트(보리·게·참기름) — 사진과 교차 사용해 고급 인쇄물 느낌

### 인터랙션 (이번 범위)
- CSS scroll-snap 풀스크린 스냅만. 애니메이션·패럴랙스 없음 (다음 단계)
- 우측 고정 도트 네비 8개 (복순도가 동일)
- 헤더는 투명 오버레이, 다크 섹션에서 색 반전

## 6. 애니메이션 기획 (차기 단계 — 디자인 확정 후 구현)

> 원칙: "이야기가 한 장씩 펼쳐진다"를 보조하는 절제된 모션만.
> 공통 스펙 — duration 0.8~1.0s, easing `cubic-bezier(.22,1,.36,1)`(ease-out), 이동거리 24~40px.
> 섹션 재진입 시 재재생(복순도가 방식). `prefers-reduced-motion` 대응 필수.
> 구현: IntersectionObserver + CSS transition/keyframes로 전부 가능 (라이브러리 불필요, GSAP 없이).

### 전 섹션 공통
- 진입 시 **챕터 라벨 → 헤드라인 → 영문 서브카피 → 본문 → 버튼** 순차 페이드업, stagger 120ms
- 도트 네비 활성 전환 0.3s, 헤더 색 반전 crossfade 0.3s

### 섹션별 시그니처 모션
| 섹션 | 효과 |
|---|---|
| S1 히어로 | 로드 시 카피 3줄 순차 페이드업(150ms 간격) + 배경 켄번즈(scale 1.06→1, 8s). 스크롤 힌트 세로선 반복 드로우. 추후 영상 교체 시 켄번즈 제거 |
| S2 브랜드 선언 | 세로쓰기 헤드라인 위→아래 마스크 리빌(붓으로 쓰이듯) + 세로 괘선 height 0→100% 드로우 + **낙관 도장 스탬프**(scale 1.3→1, 0.25s, 마지막 순서) |
| S3 매일 직접 | 사진 clip-path 커튼 오픈(우→좌), 카피 페이드업 |
| S4 재료 3단 | 컬럼별 stagger 120ms 페이드업 + 사진 내부 scale 1.08→1, 캡션 후행 |
| S5 4게의 약속 | 지붕 라인 SVG 스트로크 드로잉(1.2s) → 01→04 순차 페이드(각 150ms 간격) |
| S6 밥상 4단 | S4와 동일 문법(컬럼 stagger) — 시스템 일관성 |
| S7 공간 | 배경 패럴랙스(스크롤 속도 0.85배) + 카피 페이드업 |
| S8 초대 | 오버레이 밝기 0.9→0.8로 서서히 걷히며 카피 페이드업, 창업 문의 버튼 테두리 드로우 |

### 구현 우선순위
1. **1차 (공수 최소·효과 최대)**: 공통 순차 페이드업 + 스트립 컬럼 stagger
2. **2차 (브랜드 시그니처)**: 세로쓰기 마스크 리빌, 낙관 스탬프, 히어로 켄번즈
3. **3차 (마감 디테일)**: 지붕 SVG 드로잉(벡터화 필요), 패럴랙스, 버튼 테두리 드로우
