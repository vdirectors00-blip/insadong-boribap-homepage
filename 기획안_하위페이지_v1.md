## 1. 사이트 구조 & 네비게이션 모델

### 1-1. 전체 구조
```
HOME (index.html)  ← 감성 스토리 풀스크린 스크롤 (그대로 유지, 데스크탑 담당)
├─ 브랜드소개   brand.html      … 스토리 + 대표 인사말 + 비전 + 신뢰(인증·수출·제조)
├─ 메뉴안내     menu.html       … 전체 메뉴 + 가격 + 맛있게 먹는 법(다국어)
├─ 창업안내     franchise.html  … ⭐사업 목표 페이지. 강점 → 절차 → 비용(문의) → 문의
├─ 매장안내     store.html      … 본점 오시는 길 + 지도 + 영업정보
└─ 고객소통     news.html       … 공지·이벤트 + 온라인몰(스마트스토어) 링크아웃
```
> 파일명은 URL 깔끔함 위해 **영문 슬러그** 권장(GitHub Pages, `/brand` 형태로 접근). 최종 확정은 §5.

### 1-2. HOME vs 하위 — 역할 차별화 (중복 방지 핵심)
| 주제 | HOME(원페이지)에서 | 하위 페이지에서 |
|---|---|---|
| 브랜드 | 감성 티저(이야기 4장 스크롤) | 풀 스토리 + 대표 인사말 + **인증/수출/제조 팩트** |
| 메뉴 | 대표 4종 맛보기 | **전 메뉴·가격·설명 + 먹는 법(다국어)** |
| 매장 | "공간" 무드 한 컷 | **주소·지도·영업시간·오시는 길** |
| 창업 | 마지막 초대 한 마디 | **절차·강점·비용안내·문의 폼** |
| 구매 | 링크아웃 | 링크아웃(고객소통 안에서도) |

→ HOME은 **감정**, 하위는 **정보/행동**. 하위가 HOME 문장을 복붙하지 않는다.

### 1-3. 네비 매핑 (⚠️ 결정 필요 — §5-A)
현재 메인 GNB 5개: `브랜드 · 메뉴 · 매장 · 창업안내 · 구매하기` (지금은 **같은 페이지 앵커**로 이동).
하위 페이지가 생기면 이 링크들을 **페이지 이동**으로 바꿔야 함:

| GNB 라벨 | 현재(앵커) | 하위 도입 후(안) |
|---|---|---|
| 브랜드 | `#story-1` | `brand.html` |
| 메뉴 | `#table` | `menu.html` |
| 매장 | `#space` | `store.html` |
| 창업안내 | `#invite` | `franchise.html` |
| 구매하기 | 스마트스토어 | 스마트스토어(유지) |
| (고객소통) | 없음 | `news.html` — GNB 추가 여부 결정 필요 |

> ⚠️ GNB는 메인 `index.html` 안에 있어 **데스크탑 담당 영역**. 나는 하위 페이지들의 GNB만 이 매핑으로 구현하고, 메인 GNB 링크 변경은 **데스크탑 쪽에 제안/인계**한다(임의 수정 X).

## 2. 페이지별 상세 기획

> 각 페이지 공통 골격: **상단 헤더(재사용) → 페이지 히어로(작은 배너, 풀스크린 아님) → 콘텐츠 섹션들 → CTA → 푸터(재사용)**.
> 하위 페이지는 스냅 아님. 일반 문서형 스크롤.

### 2-1. 브랜드소개 `brand.html`
- **목적**: HOME 감성 티저를 못 본/더 알고 싶은 방문자에게 **신뢰의 근거**를 준다. (창업 검토자·B2B가 팩트 확인)
- **타깃 행동**: 스토리 공감 → 인증/실적으로 신뢰 → (창업안내로 자연 연결)
- **섹션 구조**
  1. 페이지 히어로: "인사동보리밥 이야기" + 한 줄 서브
  2. 브랜드 스토리: 우리의 이야기 6부작 요약(연평도 암꽃게·황학동 통참깨·매일 직접) — 명조 헤드 + 사진
  3. **대표 인사말**: 대표 임상균 명의 인사말(사진/서명) — ⚠️문안 필요
  4. 비전/철학: "전통 한식의 현대적 재해석", 3無(합성향료·감미료·착색료)
  5. **신뢰 지표(그래이츠)**: HACCP·FDA 시설등록 / 수산식품수출 브랜드대전 동상 / 현대백화점 온라인식품관 입점 / 인천 남동구 선정 브랜드 / 홍콩('21~)·미국('24~) 수출 / 제조–물류–매장 자체 통합
  6. CTA: "함께 걸어갈 분을 찾습니다" → 창업안내
- **콘텐츠 소스**: 우리의 이야기 PDF, 브랜드소개서_20251120.pdf, 기존 index 카피. **필요**: 대표 인사말 원문, 제조시설 hi-res 사진.

### 2-2. 메뉴안내 `menu.html`
- **목적**: 전 메뉴를 제대로 보여주고, 인사동 방문 외국인 대상 **"맛있게 먹는 법" 다국어** 제공.
- **타깃 행동**: 메뉴 탐색 → 방문 or 온라인몰 구매
- **섹션 구조**
  1. 페이지 히어로: "매일 줄 서서 먹는 밥상"
  2. **정식(보리밥+제철나물 8종 포함)**: 직화제육 / 직화불고기 / 직화주꾸미 / 고등어·삼치·갈치구이 — 사진+설명+가격
  3. 게장·곁들임: 연평도 간장게장 / 국내산 양념게장 / 육전 / 양념 LA갈비
  4. 찌개: 김치·순두부·된장
  5. **맛있게 먹는 법(다국어 KOR/ENG + 필요시 JPN/CHN)**: 간장게장 먹는 순서, 보리밥 비비는 법 — 아이콘 스텝
  6. CTA: "온라인으로 만나기" → 스마트스토어
- **콘텐츠 소스**: 브랜드소개서(메뉴/가격), 정식 촬영 사진 시리즈(사진,영상/), 상페.
  ⚠️ **가격 데이터 대조 필요**: 브랜드소개서_20251120 기준(정식 14,000 / 구이 17,000 / 게장·LA갈비 39,000 / 찌개 11,000 / 육전 12,000) vs 인수인계의 "정식 14,900~18,000" — **최신 확정가 1건 확정 필요**.
  ⚠️ 다국어 범위(영어만? 일·중 포함?) 결정 필요.

### 2-3. 창업안내 `franchise.html` ⭐(사업 목표 = 최우선)
- **목적**: 사이트 존재 이유. 단, **차분한 초대 톤**. "모집/수익" 외치지 않는다.
- **타깃 행동**: 강점 납득 → 절차 이해 → **문의(폼/전화/카톡)**
- **섹션 구조**
  1. 페이지 히어로: "같은 길을 함께 걸어가실 분을 찾습니다"(메인 초대 카피 승계)
  2. **왜 인사동보리밥인가(강점)**: 제조 인프라 보유 한식 다이닝 / 생산–물류–매장 자체 통합(상권·규모 무관 안정 운영) / 검증된 맛(인사동 본점) / HACCP·수출 이력
  3. 창업 절차: 문의 → 상담 → 상권분석 → 계약 → 오픈 (스텝 다이어그램) — ⚠️실제 절차 확인
  4. **개설 비용**: 톤 원칙상 **숫자 대신 "문의 후 안내"** 카드 + (원할 경우 항목만: 가맹비·인테리어·주방설비 등 구성 항목 표기)
  5. **창업 문의**: 폼(성함·연락처·지역·문의내용) 또는 전화 0507-1327-6086 / 카카오 — ⚠️채널 확정
  6. 신뢰 재확인: 그래이츠 로고 크레딧 + 인증 배지
- **콘텐츠 소스**: 브랜드소개서(차별점 3), 제안서. **필요**: 실제 창업 절차/비용 구성/문의 채널(폼 백엔드 or 전화 or 카톡채널).

### 2-4. 매장안내 `store.html`
- **목적**: 본점 방문 전환(오시는 길).
- **섹션 구조**
  1. 페이지 히어로: "인사동에서" / 매장 사진
  2. **지도**: 네이버/카카오 맵 임베드 + 네이버 플레이스 링크(naver.me/5b0AiKjp)
  3. 매장 정보: 주소 / 영업시간 / 휴무(일요) / 전화 / 주차·교통
  4. 오시는 길: 종각역/안국역 도보 안내
  5. (선택) 매장 갤러리
- **콘텐츠 소스**: 네이버 플레이스, 매장 사진.
  ⚠️ **주소 불일치 확정 필요**: index.html "서울 종로구 관훈동(인사동)·종각역 도보 5분" vs 메모리 "인사동7길37 지하1층, 일요휴무". 정확 주소·영업시간 1건 확정.
  ⚠️ 지도 임베드 방식(네이버 동적지도 API 키 필요 / 카카오맵 / 정적 이미지+링크) 결정.

### 2-5. 고객소통 `news.html`
- **목적**: 공지·이벤트 안내 + 온라인 구매 연결.
- **섹션 구조**
  1. 페이지 히어로: "소식"
  2. 공지·이벤트 리스트(정적 카드/리스트 — 최초엔 1~2건 or 준비중)
  3. 온라인몰 안내 + 스마트스토어 링크아웃(greatsfb)
  4. (선택) 인스타그램 피드/링크 — ⚠️핸들 확정(@ihboribob vs @alwayspirate_)
- **비고**: 정적 사이트라 공지 갱신은 수동(HTML 편집). 향후 WP 이관 시 게시판화. **최소 스펙으로**.

## 3. 공통 디자인/컴포넌트 시스템 (하위 페이지용)

### 3-1. 재사용 (style.css 그대로, 수정 없이 클래스만 사용)
- `.site-header`/`.gnb`/`.header-utils`/`.nav-toggle`/`.m-nav` (헤더·모바일 메뉴)
- `.site-footer` (푸터)
- `.btn-outline`(+`.dark`/`.slim`) (버튼)
- `:root` 토큰(색·서체 변수), `.h-headline`·`.body-copy`·`.chapter`·`.sub-en-s` (타이포)
> 하위 페이지 `<head>`에 **`style.css`와 `pages.css`를 둘 다 링크**. 단 `.panel`/스냅 클래스는 **사용 안 함**(하위는 문서형).

### 3-2. 신설 (`css/pages.css` — 내가 만드는 새 파일)
- **페이지 히어로**(작은 배너): 40~56vh, 사진 배경 + 크림 오버레이 + 중앙 명조 타이틀
- **섹션 컨테이너**: `max-width 1080px`, 넉넉한 상하 패딩(문서 리듬)
- 컴포넌트: 메뉴 카드/가격 리스트, 절차 스텝(번호 원), 정보 표(매장), 문의 폼, 지도 래퍼(반응형 16:9), 아코디언(맛있게 먹는 법 언어 토글), 신뢰 배지 행
- **반응형**: 메인과 동일 900px 브레이크포인트. 280~1440 오버플로우 0 목표.
- 헤더 색: 하위 페이지 히어로는 대체로 밝음 → 헤더 `on-light` 고정 or 자체 상태(메인 JS 스크롤스파이 불필요). 하위 페이지는 **작은 자체 JS**(햄버거 토글만) 인라인 사용.

### 3-3. 접근성/SEO
- 각 페이지 고유 `<title>`·`meta description`·OG 태그
- 시맨틱 헤딩(h1=페이지 타이틀), 이미지 alt, 폼 label
- 페이지 간 상호 링크(브레드크럼/푸터 사이트맵)

## 5. 결정 필요 사항 (기획 확정 게이트)

- **A. 네비 모델**: GNB를 앵커→페이지 이동으로 전환? 고객소통을 GNB에 넣나? (메인 GNB 변경은 데스크탑 인계)
- **B. 제작 범위/우선순위**: 5개 전부 vs 우선순위부터(권장: 창업안내 → 메뉴안내 → 매장안내 → 브랜드소개 → 고객소통)
- **C. 파일/URL 스킴**: 영문 슬러그(brand/menu/…) vs 한글
- **D. 창업 문의 채널**: 폼(백엔드 필요) vs 전화/카톡 링크
- **E. 다국어 범위**: 영어만 vs 일·중 포함
- **F. 확정 데이터 3건**: 메뉴 최신가 / 정확 주소·영업시간 / 인스타 핸들

## 6. 제안 제작 순서 (확정 후)
