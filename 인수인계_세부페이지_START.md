## 3. 작업 규칙 (이 프로젝트에서 반드시)
- **추측 금지·원자료 검증**: 팩트(메뉴·가격·주소·인증)는 반드시 `참고자료/`(브로슈어·리플렛·우리의 이야기 6부작·브랜드소개서) 원본과 대조. 없는 사실은 **지어내지 말고 placeholder**(`data-todo`)로 비워둠. v2 §1에 전수 대조 결과 정리돼 있음.
- **한 섹션/한 페이지씩 → 렌더 확인 → 커밋**(롤백 포인트). 커밋만 하고 push는 사용자 확인 후.
- **anti-AI 원칙**(v2 §9A·§10): 제네릭 아이콘폰트/이모지 금지, Lucide 1.5px 단일세트, 낙관·지붕라인·수채 재사용, 크림/우드+인장적색 1포인트, em-dash 금지, keep-all.
- **최종 판정 = 렌더**. 헤드리스 통과 ≠ 시각검증. 반드시 스크린샷 눈으로.

## 4. 디자인 시스템 (style.css 재사용 토큰)
- 색: `--cream #FAF0DC` · `--hanji #F6ECD6` · `--ink #2A2724` · `--ink-soft #4A453E` · `--seal-red #B5382A`(포인트 전용) · `--dark #1C1917` · `--white #FDFBF6`
- 폰트: `--serif`(Arita Buri 명조=헤드라인) · `--serif-en`(Fraunces=영문 서브) · `--sans`(Pretendard=본문)
- 재사용 클래스: `.site-header`/`.site-footer`, `.btn-outline`, `.chapter`(인장적색 라벨), `.seal`(낙관), `.roofline`(기와 크라운), `h3::after`(28px 인장적색 구분선), `.ing-strip`(오프셋 재료), `.menu-list`(active 인장바), 켄번즈 hover.
- **디바이스별 줄바꿈 헬퍼(이번에 추가)**: `<br class="br-d">`=PC 전용, `<br class="br-m">`=모바일 전용. 반응형 br 앞엔 **공백 필수**(숨김 시 단어 붙음 주의). 본문은 `text-wrap: pretty`로 흐르게, 헤드라인은 `balance`.
- 반응형 브레이크포인트 **900px**. 섹션 높이는 짧은 노트북(≈768) 대응 위해 **사진/여백을 svh 비례 clamp**로(메인 S5·S6에서 쓴 패턴). 목표: 어떤 화면 높이든 오버플로 0.

## 5. 실무 정보
- **로컬 서버**: `.claude/launch.json`의 `site` 설정(python http.server, `site/` 서빙) → **http://localhost:5173**. 정적 서버라 저장 즉시 반영(새로고침 Ctrl+F5).
- **렌더 검증**: playwright + 시스템 크롬(`channel="chrome"`) 헤드리스. ⚠️ 스크립트에 경로 하드코딩 시 Git Bash가 argv만 `/c/…`→`C:\…` 변환하므로 **경로는 argv로 넘길 것**(스크립트 내부 하드코딩하면 어긋남). 저해상 캡처는 reveal 페이드 애니메이션 타이밍 때문에 흐리게 잡힐 수 있음 → 대기 2s+ 후 opacity 확인.
- **배포**: public repo `vdirectors00-blip/insadong-boribap-homepage`, `git push origin main` → GitHub Pages 라이브(`https://vdirectors00-blip.github.io/insadong-boribap-homepage/`) 1~2분 반영. 루트 `index.html`이 `./site/`로 리다이렉트.
- `.gitignore`: `사진,영상/`·`참고자료/`·`*.pdf`·`*.mp4` 제외(단 `site/assets/video/*.mp4`는 포함). 기획안 `*.md`는 추적됨.
