## 1. 실행 방법

정적 사이트라 서버만 있으면 됩니다. `site/` 폴더 기준으로:

```bash
# 방법 A) Python (설치돼 있으면)
cd site
python -m http.server 5173
# → 브라우저에서 http://localhost:5173

# 방법 B) VS Code Live Server 확장으로 index.html 열기
# 방법 C) Node
npx serve site
```

> ⚠️ `file://`로 직접 열어도 대부분 보이지만, 일부 브라우저에서 로컬 폰트/이미지 경로 이슈가 있을 수 있으니 **로컬 서버 사용 권장**.

## 3. ⚠️ 폰트 (다른 PC에서 가장 먼저 확인할 것)

헤드라인 폰트로 **조선일보명조**(로컬 설치 폰트)를 쓰고 있습니다. CSS 폴백 체인:

```css
--serif: '조선일보명조', 'ChosunilboNM', '마루 부리 중간', 'MaruBuri',
         'Gowun Batang', 'Noto Serif KR', serif;
--sans:  'Pretendard Variable', Pretendard, sans-serif;
```

- **조선일보명조가 없는 PC에서는** 마루 부리 → 고운바탕(웹폰트) 순으로 폴백됩니다. 디자인 결이 달라 보일 수 있음.
- **본문 Pretendard**는 CDN(jsdelivr)에서 로드하므로 인터넷만 되면 어디서든 동일.
- **고운바탕/Noto Serif KR**는 Google Fonts CDN에서 로드됨 (index.html `<head>`).
- **배포 시 할 일**: 조선일보명조를 웹폰트로 임베드하거나, 산돌구름 웹폰트(산돌 정체 등)로 교체해야 방문자에게도 동일하게 보임. 지금은 "작업 PC에서 보기용" 상태.

> 헤드라인 폰트를 바꾸려면 `style.css`의 `--serif` 한 줄만 수정.

## 4. 페이지 구조 (8섹션 + 푸터)

풀스크린 스냅(`scroll-snap`). 우측 도트 네비 8개. 헤더는 로고(좌)·네비(중앙)·KOR|ENG(우) 3단.

| # | 섹션 id | 내용 | 레이아웃 | 배경/톤 |
|---|---|---|---|---|
| S1 | `#hero` | 히어로 "서울 인사동 골목길…" | 풀스크린 사진 + 중앙 카피 | 다크(북촌 사진) |
| S2 | `#story-1` | 이야기 하나 — 인사동 보리밥 | 좌 사진 / 우 크림(세로쓰기+낙관) | 밝음 |
| S3 | `#story-2` | 이야기 둘 — 매일 직접, 정성껏 | 좌 크림 / 우 사진 (반전) | 밝음 |
| S4 | `#story-3` | 이야기 셋 — 재료의 진심 | 상단 카피 + 하단 3단 재료 스트립(게장·장·보리) | 밝음 |
| S5 | `#story-4` | 이야기 넷 — **4게의 약속** | 지붕라인 + 4열(워터마크 넘버·구분선) + 낙관 | 밝음 |
| S6 | `#table` | 밥상 — 매일 줄 서서 먹는 밥상 | 좌 대표 대형(제육) / 우 메뉴 리스트 | 밝음 |
| S7 | `#space` | 공간 — 따뜻한, 한 끼 | 풀스크린 사진(상차림) + 중앙 카피 | 다크 |
| S8 | `#invite` | 초대 — 창업 문의 | 풀스크린 사진(한 상 탑뷰) + 웜 오버레이 | 웜다크 |
| — | 푸터 | 로고·회사정보·링크 | 독립 블록 (**스냅·도트 네비 제외**) | 다크 |

네비 메뉴: **브랜드 / 메뉴 / 매장 / 창업안내 / 구매하기**
(브랜드→#story-1, 메뉴→#table, 매장→#space, 창업안내→#invite, 구매하기→스마트스토어)

```python
from PIL import Image, ImageOps
im = ImageOps.exif_transpose(Image.open("원본.jpg"))
if im.width > 2400: im = im.resize((2400, round(im.height*2400/im.width)), Image.LANCZOS)
im.convert("RGB").save("assets/img/출력.jpg", quality=82, optimize=True, progressive=True)
```

## 8. 주요 의사결정 히스토리 (왜 이렇게 됐는지)

- **히어로**: 북촌 골목 사진(정적)으로 확정. 추후 자체 영상으로 교체 예정(레이아웃 유지). "지하 한 켠" 문구는 삭제함
- **스토리 6부작 → 4섹션 압축**: 우리의 이야기 PDF 6장을 4개 이야기 섹션으로 통합
- **"정갈한 맛" 섹션 → "4게의 약속"으로 교체**: 재료 섹션과 메시지 중복 해소 + 상세페이지 브랜드 언어("4게의 약속" = 게 pun) 통일
- **재료의 진심**: 풀스크린 단일 사진 → 게장·고추장·보리 3단 조합으로 변경
- **밥상**: 4열 풀블리드 나열 → 좌 대표 대형 + 우 메뉴 리스트로 변경 (재료 3단과 문법 겹침 해소)
- **초대**: 다크 그레인 텍스처 → 한 상 탑뷰 사진 + 웜 오버레이 (너무 어둡다는 피드백)
- **쉼표 빨강 제거**: 문장부호는 전부 먹색. 레드는 라벨·넘버·낙관·밑줄에만
- **배경색**: 상세페이지에서 실측(`#F8EEDB~#FBF0D9`)해 `--cream: #FAF0DC`로 교정. 한지 텍스처 오버레이는 상세페이지에 없어서 제거
- **로고**: 상세페이지 추출본 → AI 원본 300dpi 래스터화본으로 교체(선명도 개선). AI에 세로형 락업(2p)도 있음 — 모바일/푸터용 후보
- **푸터**: 초대 섹션 안에 있던 것을 독립 블록으로 분리 (도트 네비 대상에서 제외)
